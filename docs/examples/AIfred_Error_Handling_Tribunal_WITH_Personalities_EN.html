from sqlalchemy.exc import SQLAlchemyError, OperationalError, IntegrityError

def get_user(user_id: int) -> User | None:
    try:
        user = db.query(User).filter(User.id == user_id).first()
        if user is None:
            logger.warning(f"User {user_id} not found")
        return user
    except (OperationalError, IntegrityError) as e:
        logger.error(f"Database integrity or connection issue: {e}")
        return None
    except SQLAlchemyError as e:
        logger.error(f"Unexpected SQLAlchemy error: {e}")
        return None
If you actually need a generic handler – for example in an environment with uncontrollable third-party dependencies – then document it explicitly and use logger.critical with e!r to document the cause – but only if you're sure you can analyze the error later. Because: An error that is not visible is an error that grows – but an error that is visible is an error that can be healed. The true Tachlis is not to catch everything – but to catch only what you understand.
