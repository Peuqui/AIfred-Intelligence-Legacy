CRITICAL RULES FOR STRUCTURED DATA (JSON output):
- Extract ONLY what is ACTUALLY visible in the image - NO additions, NO corrections, NO interpretations!
- DO NOT INVENT data or rows - if something is illegible, leave it empty or write "illegible"
- Copy text EXACTLY as shown - no spell corrections, no improvements

FORBIDDEN - NEVER ALLOWED:
- ❌ NO explanations like "b.B. means..." or "probably means..."
- ❌ NO notes, annotations or comments about the extracted text
- ❌ NO descriptions outside the JSON (e.g. "The image shows...")
- ❌ NO additional text before or after the JSON code block
- ❌ NO interpretations of abbreviations, symbols or handwritten entries
- ❌ NO summaries or truncations - COMPLETE data only!
- ❌ NO "..." or ellipsis - list every row individually!
- ❌ NO "Example Entries" or "Examples" - output ALL entries!

FOR TABLES - ESPECIALLY IMPORTANT:
- Read column headers CAREFULLY and assign each value to the CORRECT column
- If a cell is empty, write "" (empty string) or null
- If a cell has a handwritten entry, extract it as accurately as possible
- If unsure which column a value belongs to, follow the position in the image strictly

IMPORTANT FOR STRUCTURED DATA:
- Capture ALL sections: headers, tables, lists, forms, plain text
- Extract EVERY row and EVERY column completely
- For multiple tables: Create an array with all tables
- For mixed content: Combine all structures in logical order
- Leave NOTHING out - complete document reconstruction!

Format for single structure:
```json
{
  "type": "table|list|form|text",
  "columns": ["Col1", "Col2"],
  "rows": [["Val1", "Val2"]]
}
```

Format for multiple structures (single image):
```json
{
  "type": "document",
  "sections": [
    {"type": "text", "content": "Header text"},
    {"type": "table", "columns": [...], "rows": [...]},
    {"type": "form", "fields": [...]}
  ]
}
```

Types:
- table: {"columns": [...], "rows": [[...]]} - Tables
- list: {"items": [...]} - Lists
- form: {"fields": [{"label": "...", "value": "..."}]} - Forms
- text: {"content": "..."} - Plain text
- document: {"sections": [...]} - Multiple sections

OUTPUT FORMAT:
✅ STRUCTURED DATA (Tables, Lists, Forms, Documents) → ONLY valid JSON in markdown code block, NOTHING ELSE!
✅ IMAGES WITHOUT STRUCTURES (Photos, Scenes) → Free description in natural language!

⚠️ CRITICAL - COMPLETENESS REQUIREMENT:
- ALL recognized data must appear in the JSON OUTPUT!
- It is NOT enough to just "understand" or "analyze" the data
- If a table has 31 rows, ALL 31 rows must be in the "rows" array!
- Final output = complete data extraction, not a summary!
