**1. Intent:**

**2. Addressee (who should respond):**
Available agents (for context only — DO NOT route based on topic):
{agent_list}

**Set an addressee ONLY when the user EXPLICITLY addresses an agent by name.**
Topic-based / expertise routing is DISABLED — the user picks their agent
manually via UI button (browser) or wake-word (Puck). Do not infer based
on the query subject (math → Codi, philosophy → Sokrates etc.).

Rules for name detection:
- Speaking TO someone = addressee ("Hey Sokrates, ...", "Codi, hilf mir").
- Speaking ABOUT someone = NOT addressee ("Was meint Sokrates?" → leer).
- Ignore grammar/spelling — focus on semantic intent.
- **Phonetic variations matter**: STT transcripts are noisy (Whisper, Vosk).
  Each agent's entry above lists ``aliases`` if present — these are common
  STT misspellings. Match user input against name AND aliases. If unsure
  whether a phonetic variant refers to an agent: prefer matching it.

If no explicit name address → leave EMPTY (use ``||`` in output). The
server falls back to the active agent (UI selection / last sticky).

**3. Language of the user text in quotes above:**
- DE: The text is in German (Deutsch)
- EN: The text is in English

Detect the language of the QUOTED USER TEXT only, not these instructions.

**4. Mode Switch (detect if the user wants to change AIfred's settings):**

The user may request a configuration change. Detect these and encode them as
comma-separated key=value pairs:

- `research=none`    → use own knowledge only, no web
- `research=quick`   → quick web search (3 results)
- `research=deep`    → deep web research (7 results)
- `research=automatik` → let AI decide automatically
- `multi=standard`   → single agent responds (default)
- `multi=tribunal`   → all three agents debate (AIfred + Sokrates + Salomo)
- `multi=symposion`  → multiple selected agents discuss
- `multi=critical_review` → Sokrates critically reviews AIfred's answer
- `multi=auto_consensus`  → agents debate until consensus
- `agent=<id>`       → set a specific agent as responder. **MUST be the
  lowercase agent ID** from the list in section 2 above (e.g. `hal`,
  `pater`, `codi`) — NEVER the display name (NOT `HAL 9000`, NOT `Pater Tuck`).

Examples of mode switch detection (works in ALL languages):
- "Start tribunal"                          → multi=tribunal
- "Démarre le tribunal"                     → multi=tribunal
- "Inicia el tribunal"                      → multi=tribunal
- "Schalt auf Tiefrecherche"                → research=deep
- "Use own knowledge only"                  → research=none
- "Solo Sokrates debería responder"         → multi=standard,agent=sokrates
- "Ich will mit Sokrates weiter reden"      → agent=sokrates
- "I want to keep talking to HAL"           → agent=hal      (lowercase id, NOT "HAL 9000")
- "Schalt um auf HAL 9000"                  → agent=hal      (lowercase id, NOT "hal 9000")
- "Bleib bei Pater Tuck"                    → agent=pater    (lowercase id, NOT "Pater Tuck")
- "Wie geht's?"                             → EMPTY (no mode switch)

Detecting persistent agent switch:
- If the user says they want to CONTINUE talking to / discussing with / stay with a specific agent,
  that is an `agent=<id>` switch (persistent). This is different from a one-time addressee (field 2).
- One-time: "Sokrates, erkläre mir X" → Addressee only, no mode switch
- Persistent: "Ich möchte mit Sokrates weiter diskutieren" → agent=sokrates mode switch

Only fill this field if the user EXPLICITLY requests a mode/configuration change.
Questions about modes ("What is Tribunal mode?") are NOT mode switches — leave EMPTY.

**5. Remaining Query:**

If the user combined a mode switch with an actual question, extract the question part
here (in the user's original language, verbatim). If the message is ONLY a mode switch
command, leave this EMPTY.

Examples:
- "Start tribunal and discuss climate change"     → Remaining: "discuss climate change"
- "Schalt auf Tiefrecherche und such mir Infos zu X" → Remaining: "such mir Infos zu X"
- "Start tribunal"                                 → Remaining: EMPTY
- "Wie geht's?"                                    → Remaining: EMPTY (no mode switch at all)

**Output format - STRICT SCHEMA:**

Return EXACTLY this format: INTENT|ADDRESSEE|LANGUAGE|MODE_SWITCH|REMAINING

Rules:
- INTENT must be: FACTUAL, CREATIVE, or MIXED (uppercase)
- ADDRESSEE must be: one of the agent IDs listed above, or EMPTY (lowercase, no quotes)
- LANGUAGE must be: DE or EN (uppercase)
- MODE_SWITCH: comma-separated key=value pairs, or EMPTY
- REMAINING: the cleaned question text without the mode command, or EMPTY
- Use | as separator (pipe character) between fields
- Empty fields: leave blank between pipes (example: FACTUAL||DE||)
- No explanations, no extra text, ONLY the five fields

Examples of complete output:
- "Wie geht's?"                          → FACTUAL||DE||
- "Start tribunal"                       → FACTUAL||EN|multi=tribunal|
- "Starte Tribunal und diskutiert X"     → FACTUAL||DE|multi=tribunal|diskutiert X
