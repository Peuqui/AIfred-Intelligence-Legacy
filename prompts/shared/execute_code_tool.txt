Execute Python code in a secure sandbox to perform calculations, data analysis, or generate visualizations. Use this tool when the user asks for: mathematical computations, statistical analysis, data processing, chart/plot generation, simulations, or any task that benefits from actual code execution rather than estimation. Available libraries: math, statistics, numpy, pandas, matplotlib, scipy, sklearn, seaborn, plotly. The sandbox has no network access and a 30-second time limit. Always print() results you want to return. For INTERACTIVE visualizations (3D plots, animations, dashboards): use plotly and save with fig.write_html("output.html", include_plotlyjs=True). For static plots: use matplotlib (images are captured automatically). For HTML/JS apps: write to "output.html". IMPORTANT: When generating HTML with Canvas or visualizations, always use width:100% and height:100vh so the content fills the entire display area — never use fixed pixel sizes. Set body margin and padding to 0. IMPORTANT: All plots and HTML outputs are automatically displayed in the chat. After calling this tool, do NOT attempt to display images yourself — no base64, no data URIs, no img tags. Just describe the result in text.

FILE ACCESS: The user's documents are available under the relative path "documents/" (e.g. open("documents/Polizei/foo.txt")) — READ-ONLY, attempts to open them in write mode will fail with PermissionError. If you need to persist output into the user's documents directory, use the separate `execute_code_write` tool (requires elevated permission). Otherwise write into the current directory (e.g. open("result.txt", "w")) — those files are ephemeral but can be returned via print() or HTML/image output.
