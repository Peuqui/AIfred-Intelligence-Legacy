Execute Python code in a secure sandbox WITH WRITE ACCESS to the user's documents directory. Use this tool ONLY when the task requires modifying, creating, or deleting files inside documents/ (e.g. splitting a PDF, saving a generated report, batch-renaming files, consolidating duplicate text files). For purely analytical work (calculations, plots, read-only processing), use the regular `execute_code` tool instead — it has the same capabilities minus write access and is preferred. Available libraries: math, statistics, numpy, pandas, matplotlib, scipy, sklearn, seaborn, plotly. The sandbox has no network access and a 30-second time limit. Always print() results you want to return. For INTERACTIVE visualizations (3D plots, animations, dashboards): use plotly and save with fig.write_html("output.html", include_plotlyjs=True). For static plots: use matplotlib (images are captured automatically). For HTML/JS apps: write to "output.html". IMPORTANT: When generating HTML with Canvas or visualizations, always use width:100% and height:100vh so the content fills the entire display area — never use fixed pixel sizes. Set body margin and padding to 0. IMPORTANT: All plots and HTML outputs are automatically displayed in the chat. After calling this tool, do NOT attempt to display images yourself — no base64, no data URIs, no img tags. Just describe the result in text.

FILE ACCESS: The user's documents are available READ-WRITE under the relative path "documents/" (e.g. open("documents/result.txt", "w")). Changes persist on disk. Be conservative: do not delete or overwrite files the user did not explicitly ask about. When in doubt, write new files rather than modifying existing ones.
