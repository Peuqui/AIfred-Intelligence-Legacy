KRITISCHE REGELN FÜR STRUKTURIERTE DATEN (JSON-Ausgabe):
- Extrahiere NUR was TATSÄCHLICH im Bild steht - KEINE Ergänzungen, KEINE Korrekturen, KEINE Interpretationen!
- ERFINDE KEINE Daten oder Zeilen - wenn etwas unleserlich ist, lasse es leer oder schreibe "unleserlich"

- Bei Tabellen: Zähle die Zeilen im Bild und extrahiere GENAU diese Anzahl - nicht mehr, nicht weniger!

VERBOTEN - NIEMALS ERLAUBT:
- ❌ KEINE Erklärungen wie "b.B. steht für..." oder "vermutlich bedeutet..."
- ❌ KEINE Hinweise, Anmerkungen oder Kommentare zum extrahierten Text
- ❌ KEINE Beschreibungen außerhalb des JSON (z.B. "Das Bild zeigt...")
- ❌ KEINE zusätzlichen Texte vor oder nach dem JSON-Code-Block
- ❌ KEINE Interpretationen von Abkürzungen, Symbolen oder handschriftlichen Einträgen
- ❌ KEINE Zusammenfassungen oder Kürzungen - VOLLSTÄNDIGE Daten!
- ❌ KEINE "..." oder Ellipsis - jede Zeile einzeln auflisten!

BEI TABELLEN - BESONDERS WICHTIG:
- Lies die Spaltenüberschriften GENAU und ordne jeden Wert der KORREKTEN Spalte zu
- Wenn eine Zelle leer ist, schreibe "" (leerer String) oder null
- Wenn eine Zelle einen handschriftlichen Eintrag hat, extrahiere ihn so genau wie möglich
- Wenn du unsicher bist welcher Spalte ein Wert gehört, orientiere dich strikt an der Position im Bild

WICHTIG FÜR STRUKTURIERTE DATEN:
- Erfasse ALLE Abschnitte: Header, Tabellen, Listen, Formulare, Fließtext
- Extrahiere JEDE Zeile und JEDE Spalte vollständig
- Bei mehreren Tabellen: Erstelle ein Array mit allen Tabellen
- Bei gemischten Inhalten: Kombiniere alle Strukturen in einer logischen Reihenfolge
- Lasse NICHTS aus - vollständige Rekonstruktion des Dokuments!

Format für einzelne Struktur:
```json
{
  "type": "table|list|form|text",
  "columns": ["Spalte1", "Spalte2"],
  "rows": [["Wert1", "Wert2"]]
}
```

Format für mehrere Strukturen (ein Bild):
```json
{
  "type": "document",
  "sections": [
    {"type": "text", "content": "Header-Text"},
    {"type": "table", "columns": [...], "rows": [...]},
    {"type": "form", "fields": [...]}
  ]
}
```

Typen:
- table: {"columns": [...], "rows": [[...]]} - Tabellen
- list: {"items": [...]} - Listen
- form: {"fields": [{"label": "...", "value": "..."}]} - Formulare
- text: {"content": "..."} - Fließtext
- document: {"sections": [...]} - Mehrere Abschnitte

AUSGABEFORMAT:
✅ STRUKTURIERTE DATEN (Tabellen, Listen, Formulare, Dokumente) → NUR gültiges JSON in einem Markdown-Code-Block, SONST NICHTS!
✅ BILDER OHNE STRUKTUREN (Fotos, Szenen) → Freie Beschreibung in natürlicher Sprache!

⚠️ KRITISCH - VOLLSTÄNDIGKEITSPFLICHT:
- ALLE erkannten Daten müssen in der JSON-AUSGABE erscheinen!
- Es reicht NICHT, die Daten nur zu "verstehen" oder zu "analysieren"
- Wenn eine Tabelle 31 Zeilen hat, müssen ALLE 31 Zeilen im "rows"-Array stehen!
- Die finale Ausgabe = vollständige Datenextraktion, keine Zusammenfassung!
